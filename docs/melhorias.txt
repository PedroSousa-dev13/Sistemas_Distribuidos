2) MELHORIAS FUNCIONAIS ADICIONADAS
-----------------------------------
2.1) Novo cliente DataStreamClient
- Foi criado um cliente adicional para leitura de ficheiro CSV e envio continuo
  de medicoes para a Gateway.
- Este cliente reutiliza o SensorClient para manter consistencia de protocolo.

2.2) Streaming controlado por tecla
- O DataStreamClient so comeca o envio apos confirmacao do utilizador.
- Vantagem: melhor controlo durante demonstracao em aula.

2.3) Resolucao robusta de caminho do CSV
- Implementada logica para localizar o CSV em diferentes diretorios de execucao
  (VS Code e Visual Studio).
- Evita erro recorrente de "ficheiro nao encontrado" em debug.

4) ARQUITETURA E CONCORRENCIA (ESTADO FINAL)
---------------------------------------------
4.1) Gateway
- Recebe mensagens dos sensores.
- Enfileira DATA para processamento controlado.
- Encaminha para servidor e propaga ACK ao sensor.

4.2) Servidor
- Aceita ligacoes concorrentes de gateways.
- Faz persistencia com mutex por tipo de ficheiro.
- Garante exclusao mutua na escrita.

4.3) Sensor e DataStreamClient
- Sensor: operacao manual via menu.
- DataStreamClient: operacao automatica por dataset.
- Ambos seguem o mesmo protocolo de registo e envio.

6) COMANDOS OFICIAIS PARA DEMONSTRACAO
--------------------------------------
Build:
- dotnet build SharedProtocol.sln -c Debug

Servidor:
- dotnet run --project src/Servidor/Servidor.csproj -- 6000

Gateway:
- dotnet run --project src/Gateway/Gateway.csproj -- 5000 127.0.0.1:6000 ./sensores.csv

Sensor manual:
- dotnet run --project src/Sensor/Sensor.csproj -- 127.0.0.1 5000 sensor-01

DataStreamClient:
- dotnet run --project src/DataStreamClient/DataStreamClient.csproj -- 127.0.0.1 5000 dados/stream_dados.csv
